Sudo Git Clone - Permission Denied

Posted on 2020-12-07

以 sudo 執行 git clone

問題發生在使用 Server 從遠端 Repo 下載程式時，
git clone 跳出錯誤訊息，顯示我目前沒有權限存取：

而我已經在 Gitlab repo 上設定好了 Deploy key，Gitlab 應該可以辨識才對，

我是以 ssh 的方式去 clone 專案的，因此先從 ssh 去嘗試是否正常：

ssh 能夠正常連到 Gitlab，並知道這是由誰加入的 deploy key，那麼問題可能不在於權限。

於是開始思考前面 git clone 跟 ssh 之間的差異，發現一個可能性
我是用 sudo 的方式做 git clone，因為這個資料的權限設定是擁有者才能寫入，

再一次嘗試

找到了問題的關鍵點是由 sudo 導致的，sudo 會以 root 身份執行，
ssh 的 private key 會從 /home/user/.ssh 裡頭拿出來的，而 sudo ssh 拿取的路徑卻改成了 /root/.ssh，根本不是使用原本的設定好的 ssh key，因此當然沒有權限。
